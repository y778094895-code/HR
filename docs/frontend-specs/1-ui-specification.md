# 1. مواصفات واجهة المستخدم (UI Specification)

## نظرة عامة على التصميم (Design Overview)
- **النمط العام (Overall Style):** واجهة تحليلية حديثة مبنية على البطاقات (Card-based). مساحات نظيفة، هرمية واضحة، ظلال خفيفة، وحواف دائرية (Rounded corners).
- **الألوان (Colors):**
  - **الخلفيات:** بيضاء / رمادي فاتح (White / Light Gray)
  - **النصوص:** رمادي داكن (Dark Gray)
  - **ألوان التمييز (Accents):** أزرق للإجراءات والروابط الأساسية، أحمر/كهرماني للتحذيرات والمخاطر.
- **إمكانية الوصول (Accessibility):** تباين عالٍ، نصوص مقروءة بوضوح، تركيز لوحة المفاتيح (Keyboard focus)، وتسميات (ARIA labels) لجميع الأزرار والأدراج والتنقل.
- **التخطيط الافتراضي (Default Layout):** من اليمين إلى اليسار (RTL) بشكل افتراضي ليدعم اللغة العربية، مع إمكانية التبديل إلى LTR.

## الخطوط (Typography)
- **النصوص العادية (UI Sans-serif):** `Inter` أو `Plus Jakarta Sans` مع خط بديل عربي: `system-ui, -apple-system, "Noto Kufi Arabic", "Cairo", "Tajawal"`.
- **الأرقام والمقاييس (Monospace):** `JetBrains Mono` أو `IBM Plex Mono` للمقاييس (Metrics)، والنسب المئوية، والمعرفات، وتصديرات البيانات.
- **تنسيق الأرقام:** `tabular-nums` لمحاذاة دقيقة في الجداول وبطاقات مؤشرات الأداء (KPIs).

## مواصفات الصفحات (Screen-by-Screen UI)

### 1) تسجيل الدخول `/login`
- **الواجهة:**
  - حقول الإدخال: بريد إلكتروني (Email) وكلمة مرور (Password) مع تحقق من صحة المدخلات.
  - زر الدخول: يحتوي على حالة تحميل (Loading State) تعرض النص "جاري تسجيل الدخول..." أو (Logging in...).
  - تلميح ببيانات الدخول الافتراضية أسفل النموذج.
- **السلوك:**
  - عند النجاح: توجيه إلى `/dashboard`.
  - عند الفشل: ظهور منطقة مخصصة لعرض رسالة الخطأ.
  - إذا كان المستخدم مسجلاً بالفعل: توجيه تلقائي إلى `/dashboard`.

### 2) لوحة التحكم الرئيسية `/dashboard`
- **رأس الصفحة:** العنوان + عبارة الترحيب.
- **أزرار التصدير (Export):** تصدير كـ PDF وتصدير كـ Excel (تنزيلات وهمية Mock).
- **زر الإجراء السريع (Quick Action):** موجود كواجهة مستخدم فقط (No-op - يطبع بالكونسول).
- **شريط الفلاتر (FiltersBar):** فلترة بالفترة الزمنية والقسم (مرئي للمسؤول/المدير)، دون شريط بحث.
- **بطاقات الأداء (KPI Cards):** (غير قابلة للنقر) وتشمل:
  1. إجمالي الموظفين.
  2. الاستقالات عالية الخطورة.
  3. نسبة إكمال التدريب.
  4. متوسط الأداء.
- **الرسوم البيانية:**
  - بطاقة "اتجاهات الأداء (Performance Trends)": عنصر نائب (Placeholder).
  - قائمة "النشاط الأخير (Recent Activity)": بيانات وهمية (Mock).

### 3) التدخلات `/dashboard/interventions`
- **ملخص الأثر (ImpactSummaryChart):** بطاقات إحصائية + رسم بياني شريطي (Bar Chart).
- **الحاويات بعلامات التبويب (TabsContainer):**
  - **علامة (التدخلات):** قائمة بطاقات توضح حالة التدخل والإجراءات:
    - (قيد الانتظار PENDING) -> زر "البدء" -> يتحول إلى ACTIVE.
    - (نشط ACTIVE) -> زر "إكمال" -> يتحول إلى COMPLETED.
  - **علامة (توصيات الذكاء الاصطناعي):** قائمة توصيات من `/api/recommendations`.
    - تحتوي كل بطاقة على: شارة النوع (Badge)، نسبة الثقة، العنوان، الوصف، و"منطق الذكاء الاصطناعي".
    - أزرار القرار (في حالة generated): "رفض (Reject)" أو "تطبيق (Apply)" والتي تتصل بـ API مخصصة وتقوم بإظهار إشعار (Toast) بالنجاح/الفشل.

### 4) الموظفون `/dashboard/vo-employees`
- **رأس الصفحة:** العنوان + وصف الواجهة.
- **زر (إضافة موظف):** مرئي (No-op - بالكونسول فقط).
- **شريط البحث:** فلترة نصية عبر إدخال مصطلح البحث.
- **جدول البيانات (DataTable):**
  - الأعمدة: الموظف (صورة شخصية، الاسم، البريد)، القسم، الدور، شارة الحالة، والإجراءات.
  - الإجراءات المتاحة لكل صف: "عرض (View)" (No-op) و "حذف" (وظيفية تقوم بطلب الحذف).

### 5) إدارة المستخدمين `/dashboard/users` (لأدوار admin / super_admin)
- **زر (إضافة مستخدم):** مرئي (No-op - بالكونسول فقط).
- **جدول البيانات:**
  - إجراء "تعديل (Edit)" (No-op).
  - إجراء "حذف (Delete)" (وظيفية وتقوم بطلب الحذف).

### 6) تحليل الأثر `/dashboard/impact`
- **الواجهة:**
  - بطاقات الأداء (KPIs).
  - مخطط مساحي لاتجاه الأثر (Impact Trend - Area Chart).
  - قائمة مخرجات توضح "أعلى التوزيعات" (Top Outcome Distribution - Progress list).
  - لا توجد إجراءات تنقل داخلية هنا.

### 7) الأداء `/dashboard/performance`
- **علامات التبويب (متزامنة مع URL `?view=`):**
  - **النظرة العامة:** بطاقات KPI، رسوم بيانية فارغة كعنصر نائب ("قريباً").
  - **الموظفون:**
    - شريط الفلاتر (بحث + القسم).
    - جدول بيانات يفتح "درج جانبي (SideDrawer)" عند النقر على أي صف.
    - إجراءات الدرج: "بدء التقييم" (No-op)، "بدء خطة تحسين PIP" (للمسؤول/المدير عند تقييم أقل من 3) (No-op).
  - **التقييمات:** عنصر نائب ("قريباً").
- **التصدير:** تحميل PDF/CSV (Mock).

### 8) الاستقالات `/dashboard/attrition`
- **علامات التبويب (متزامنة مع URL `?view=`):**
  - **النظرة العامة:** بطاقات KPI (تشمل التكلفة التقديرية)، ورسوم بيانية (Placeholder).
  - **تحليل المخاطر (XAI):**
    - شريط الفلاتر (بحث + القسم).
    - قائمة المخاطر مع زر (تحليل Analyze). النقر يفتح الدرج الجانبي.
    - الدرج الجانبي (RiskDetailDrawer) يعرض: احتمالية الخطر، مستوى الثقة، بطاقات المحركات الأساسية للخطر، رسم بياني نائبي، وتوصيات. زر "تسجيل تدخل (Log Intervention)" (No-op).
  - **سجل التدخلات:** عنصر نائب ("قريباً").
- **التصدير:** تحميل PDF/CSV (Mock).

### 9) التدريب `/dashboard/training`
- **علامات التبويب (متزامنة مع URL `?view=`):**
  - **فجوات المهارات:** بطاقات KPI، خريطة حرارية (Placeholder)، قائمة بالفجوات الحرجة.
  - **البرامج والاعتمادات:**
    - شريط فلاتر (بحث).
    - جدول البرامج. للبرامج المعلقة (للمدير/المسؤول): "قبول/رفض" (No-op).
    - النقر على الصف يفتح درج الإجراءات "قبول/رفض" (No-op).
  - **الفعالية والعائد (ROI):** عنصر نائب ("قريباً").
- **التصدير:** تحميل PDF/CSV (Mock).

### 10) العدالة `/dashboard/fairness` (مسؤول فقط)
- **علامات التبويب (متزامنة مع URL `?view=`):**
  - **نظرة عامة على المساواة:** بطاقات KPI، رسوم بيانية (Placeholder).
  - **التحليل الديموغرافي:** شريط فلاتر، جدول إحصائيات، ولافتة معلومات (Info banner).
  - **توصيات الذكاء الاصطناعي:** بطاقات توصيات مع زر "تطبيق الإجراء (Implement Action)" (No-op).
- **التصدير:** تحميل PDF/CSV (Mock).

### 11) التقارير `/dashboard/reports`
- **علامات التبويب (`?view=`):**
  - **القوالب السريعة:** شبكة القوالب مع زر "تشغيل الآن (Run Now)" (No-op - يقوم بتحديث الحالة داخلياً لتوجيهها لمركز التنزيل أحياناً ولكنه لا يعمل تغيير واضح).
  - **المُنشئ المخصص:** تحديد مصدر البيانات، الفترة الزمنية، والصيغة، ثم "إنشاء تقرير (Generate)" (No-op).
  - **مركز التنزيل:** لافتة توضح الجيل غير المتزامن، وجدول للمهام (استعلام النبض Polling موجود). التنزيل مفعل فقط للمهام المكتملة.

### 12) الإعدادات `/dashboard/settings`
- **علامات التبويب (`?tab=`):**
  - **عام:** اسم الشركة، لغة العرض (إنكليزي/عربي). زر الحفظ (No-op).
  - **المستخدمون:** زر "دعوة مستخدم" (No-op)، جدول فارغ (Placeholder).
  - **الصلاحيات:** بطاقات أدوار قابلة للنقر (No-op).
  - **التكاملات:** أربطة لـ Slack / Google (No-op).
  - **الأمان:** سياسة كلمة المرور، "تهيئة المصادقة الثنائية 2FA" (No-op).
  - **المظهر:** خيارات النسق والمظهر (No-op).

### 13) المساعدة `/dashboard/help`
- **علامات التبويب (`?view=`):**
  - **قاعدة المعرفة:** بحث يفلتر المقالات محلياً، فئات ومقالات قابلة للنقر دون وجود مسار مفصل (No-op للمسارات).
  - **تذاكر الدعم:** زر لفتح درج جانبي، إرسال تذكرة يضاف لقائمة الحالة المحلية، عرض تذكرة (No-op).
  - **اتصل بالدعم:** معلومات اتصال ثابتة.
